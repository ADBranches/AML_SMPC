# Audit Traceability

## Traceability path
The prototype supports the following traceability chain:

transaction -> pseudonymization -> screening -> proof generation -> proof verification

## Stored audit evidence
- transaction submission event
- pseudonymization event details
- sender screening result
- receiver screening result
- proof artifact records
- proof verification result

## Why this matters
This traceability path supports the Recommendation 11-style record reconstruction story in the report and gives regulators or evaluators a coherent compliance timeline without exposing raw sensitive payloads.
