# FATF Mapping

## Prototype Alignment
This prototype supports compliance-oriented evidence generation aligned to:

- **FATF Recommendation 10** — customer due diligence / KYC-related assertion
- **FATF Recommendation 11** — record keeping / audit linkage assertion
- **FATF Recommendation 16** — payment-transparency field inclusion assertion

## What the prototype demonstrates
- transaction intake with privacy-preserving handling
- private sanction screening
- encrypted amount processing
- rule-specific proof artifact generation
- proof verification without exposing raw transaction payload

## Important boundary
This prototype does **not** claim full legal or supervisory certification of FATF compliance.
It demonstrates aligned controls and verifiable evidence generation in an MVP research / product prototype.
