# FINAL-5 Presentation Rehearsal Guide

## Purpose

This guide is used to rehearse the final AML SMPC project defense before delivery.

The goal is to present the project as a complete privacy-preserving AML compliance prototype, not merely a collection of services.

## Core Story

The system demonstrates how AML workflows can be made:

- privacy-preserving;
- role-governed;
- regulator-verifiable;
- auditable;
- suitable for a multi-bank research prototype.

## Correct Research Framing

The project should be explained as:

```text
Bank A, Bank B, and Bank C participate in privacy-preserving SMPC-style screening.
The regulator does not participate as a raw-input SMPC party.
The regulator verifies downstream proof and audit evidence.
```
